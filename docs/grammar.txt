===========================================================================
                BEEBO LANGUAGE — KS GRAMMAR
             (LL(1) factored grammar and parser table)
===========================================================================

ORIGINAL GRAMMAR (before left-recursion elimination)
----------------------------------------------------
Program        → StmtList
StmtList       → Stmt ; StmtList | Stmt
Stmt           → AssignStmt | IfStmt | WhileStmt | ForStmt 
                | InputStmt | OutputStmt | Block
                | FuncDef | ReturnStmt | SchemaDef
Block          → { StmtList }
AssignStmt     → id AssignRest
AssignRest     → = Expr | [ Expr ] AssignArr
                | ( ArgList ) | . id AssignRest
AssignArr      → = Expr | [ Expr ] = Expr | ε
IfStmt         → if ( Cond ) Stmt ElsePart
ElsePart       → else Stmt | ε
WhileStmt      → while ( Cond ) Stmt
ForStmt        → for ( AssignStmt ; Cond ; AssignStmt ) Stmt
InputStmt      → input LValue
OutputStmt     → output OutputArg
OutputArg      → Expr | string
FuncDef        → func id ( ParamList ) Block
ParamList      → id ParamListTail | ε
ParamListTail  → , id ParamListTail | ε
ReturnStmt     → return Expr
SchemaDef      → schema id { FieldList }
FieldList      → id FieldRest
FieldRest      → ; id FieldRest | ε
FieldInitList  → id : Expr FieldInitTail
FieldInitTail  → , id : Expr FieldInitTail | ε
LValue         → id LValueRest
LValueRest     → [ Expr ] LValueArr | ε
LValueArr      → [ Expr ] | ε
Cond           → CondTerm CondRest
CondRest       → || CondTerm CondRest | ε
CondTerm       → CondFact CondTermRest
CondTermRest   → && CondFact CondTermRest | ε
CondFact       → Expr Relop Expr | ! CondFact | ( Cond )
Expr           → Term ExprTail
ExprTail       → + Term ExprTail | - Term ExprTail | ε
Term           → Factor TermTail
TermTail       → * Factor TermTail | / Factor TermTail | ε
Factor         → ( Expr ) | int | real | id FactorRest 
                | FuncName ( Expr ) | - Factor | string
FactorRest     → [ Expr ] FactorArr | ( ArgList )
                | . id FactorRest | { FieldInitList } | ε
FactorArr      → [ Expr ] | ε
Relop          → < | > | <= | >= | == | !=
FuncName       → sqrt | exp | log | sin | cos | abs
ArgList        → Expr ArgListTail | ε
ArgListTail    → , Expr ArgListTail | ε


LL(1) FACTORED GRAMMAR USED BY THE PARSER
(after left recursion elimination and left factoring)
----------------------------------------------------
This grammar is the implementation grammar used by the predictive
parser and parse table. It is intentionally factored into helper
nonterminals such as ExprTail and TermTail. Productions beginning
with helper nonterminals are expanded through FIRST/FOLLOW table
entries during parsing.

  Program → StmtList EOF                                       (0)

 StmtList → Stmt StmtListTail                                  (1)
StmtListTail → ; Stmt StmtListTail                               (2)
StmtListTail → ε                                                (3)

     Stmt → AssignStmt                                          (4)
     Stmt → IfStmt                                              (5)
     Stmt → WhileStmt                                           (6)
     Stmt → ForStmt                                             (7)
     Stmt → InputStmt                                           (8)
     Stmt → OutputStmt                                          (9)
     Stmt → Block                                              (10)

    Block → { StmtList }                                       (11)

AssignStmt → id AssignRest                                      (12)
AssignRest → = Expr                                             (13)
AssignRest → [ Expr ] AssignArr                                 (14)
AssignArr → = Expr                                              (15)
AssignArr → [ Expr ] = Expr                                     (16)
AssignArr → ε                                                  (17)

  IfStmt → if ( Cond ) Stmt ElsePart                            (18)
ElsePart → else Stmt                                            (19)
ElsePart → ε                                                   (20)

WhileStmt → while ( Cond ) Stmt                                 (21)

 ForStmt → for ( AssignStmt ; Cond ; AssignStmt ) Stmt          (22)

InputStmt → input LValue                                        (23)
OutputStmt → output OutputArg                                   (24)
OutputArg → Expr                                                (25)

  LValue → id LValueRest                                        (26)
LValueRest → [ Expr ] LValueArr                                 (27)
LValueArr → [ Expr ]                                            (28)
LValueArr → ε                                                  (29)
LValueRest → ε                                                 (30)

    Cond → CondTerm CondRest                                     (31)
 CondRest → || CondTerm CondRest                                 (32)
 CondRest → ε                                                  (33)

CondTerm → CondFact CondTermRest                                (34)
CondTermRest → && CondFact CondTermRest                         (35)
CondTermRest → ε                                               (36)

CondFact → Expr Relop Expr                                      (37)
CondFact → ! CondFact                                           (38)
CondFact → ( Cond )                                             (39)

    Expr → Term ExprTail                                        (40)
 ExprTail → + Term ExprTail                                     (41)
 ExprTail → - Term ExprTail                                     (42)
 ExprTail → ε                                                  (43)

    Term → Factor TermTail                                      (44)
TermTail → * Factor TermTail                                    (45)
TermTail → / Factor TermTail                                    (46)
TermTail → ε                                                   (47)

  Factor → ( Expr )                                             (48)
  Factor → int                                                  (49)
  Factor → real                                                 (50)
  Factor → id FactorRest                                        (51)
  Factor → FuncName ( Expr )                                    (52)
  Factor → - Factor                                             (53)
  Factor → string                                               (54)

FactorRest → [ Expr ] FactorArr                                 (55)
FactorArr → [ Expr ]                                            (56)
FactorArr → ε                                                  (57)
FactorRest → ε                                                 (58)

   Relop → <                                                    (59)
   Relop → >                                                    (60)
   Relop → <=                                                   (61)
   Relop → >=                                                   (62)
   Relop → ==                                                   (63)
   Relop → !=                                                   (64)

FuncName → sqrt                                                 (65)
FuncName → exp                                                  (66)
FuncName → log                                                  (67)
FuncName → sin                                                  (68)
FuncName → cos                                                  (69)
FuncName → abs                                                  (70)

OutputArg → string                                              (71)

     Stmt → FuncDef                                             (72)
     Stmt → ReturnStmt                                          (73)

  FuncDef → func id ( ParamList ) Block                         (74)

ParamList → id ParamListTail                                    (75)
ParamList → ε                                                   (76)
ParamListTail → , id ParamListTail                              (77)
ParamListTail → ε                                               (78)

ReturnStmt → return Expr                                        (79)

FactorRest → ( ArgList )                                        (80)

  ArgList → Expr ArgListTail                                    (81)
  ArgList → ε                                                   (82)
ArgListTail → , Expr ArgListTail                                (83)
ArgListTail → ε                                                 (84)

AssignRest → ( ArgList )                                        (85)

     Stmt → SchemaDef                                           (86)

SchemaDef → schema id { FieldList }                             (87)

FieldList → id FieldRest                                        (88)
FieldRest → ; id FieldRest                                      (89)
FieldRest → ε                                                   (90)

AssignRest → . id AssignRest                                    (91)

FactorRest → . id FactorRest                                    (92)

FactorRest → { FieldInitList }                                  (93)

FieldInitList → id : Expr FieldInitTail                         (94)
FieldInitList → ε                                               (95)
FieldInitTail → , id : Expr FieldInitTail                       (96)
FieldInitTail → ε                                               (97)

FuncName → string                                               (98)
FuncName → real                                                 (99)
FuncName → integer                                             (100)

AssignArr2 → = Expr                                            (101)
AssignArr2 → ε                                                 (102)


NOTES:
- Numbers in parentheses are production IDs used in the parse table
- The grammar is LL(1) compatible after left factoring
- Expression precedence: - (unary) > *, / > +, -
- Operators *, / have higher priority than +, -
- Parentheses can override default precedence
- The grammar is stored together with a predictive parse table in src/grammar.ml
- Epsilon productions are used for optional tails and empty argument lists
- Schema literals use colon/comma syntax: Point{x: 10, y: 20}
